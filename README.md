# Задание 1. Кеширование

Кеширование решает ряд проблем производительности и масштабируемости:

Высокая нагрузка на базу данных. Частые запросы к БД создают нагрузку. Кеширование часто запрашиваемых данных снижает число обращений к БД.

Медленный отклик приложения. Получение данных из кеша (в оперативной памяти) быстрее, чем из БД или внешних API.

Высокая стоимость вызовов внешних API. Многие API имеют лимиты и тарифицируются по числу запросов. Кеширование результатов сокращает количество вызовов.

Повторяющиеся ресурсоёмкие вычисления. Результаты сложных вычислений (агрегации, обработки изображений) можно кешировать и переиспользовать.

Пиковые нагрузки. Кеш служит буфером, сглаживая скачки трафика и защищая основные хранилища данных.

Зависимость от доступности внешних сервисов. При недоступности внешнего API приложение может отдавать данные из кеша.

Избыточная передача данных по сети. Кеширование на стороне клиента (браузера) или прокси‑сервера (CDN) сокращает объём передаваемых данных.

Низкая масштабируемость. Без кеширования добавление серверов не сильно улучшает производительность — все они обращаются к одной БД. Кеш распределяет нагрузку.
